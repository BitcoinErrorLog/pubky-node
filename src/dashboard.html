User Guide

Everything you need to know about running and using Pubky Node.

🌐

Mainline DHT Node

Pubky Node runs a full Mainline DHT node using the BEP44 protocol. This connects you to a network of millions of peers, enabling decentralized DNS record storage and retrieval without any central server.

  • Automatic peer discovery and routing table population
  • Server mode when not firewalled for full network participation
  • Configurable UDP port via --dht-port or [dht] port in config
📡

PKARR Relay

The relay provides an HTTP API for publishing and resolving signed DNS packets. Any PKARR-compatible client can use your relay to interact with the DHT.

  • PUT /pkarr/:key — publish a signed packet
  • GET /pkarr/:key — resolve a key from DHT
  • Built-in rate limiting (2 req/s default, burst 10)
  • Persistent LMDB cache for fast repeat lookups
🔐

DNS Publisher

Sign and publish DNS records to the DHT using Ed25519 secret keys. Records are automatically re-published at a configurable interval to stay alive on the network.

  • Supports A, AAAA, CNAME, TXT, NS, and HTTPS records
  • Secret keys can be inline or loaded from files (Docker secrets compatible)
  • Automatic retry with exponential backoff on publish failures
  • Configure via [publisher] section in config.toml
🔍

Key Explorer

Look up any public key to view its DNS records stored on the DHT. Paste a 52-character z-base-32 encoded public key and see the full record set.

  • Supports all record types: A, AAAA, CNAME, TXT, NS, HTTPS, SVCB
  • Shows packet metadata: compressed size, age, record count
  • Search history stored locally in your browser
  • Also available via API: GET /api/resolve/:key
🌐

HTTP Proxy

Local proxy on port 9091 that serves .pkarr user profiles directly in your browser. Points any .pkarr hostname at 127.0.0.1 and fetches the user's profile from their homeserver.

  • One-click /etc/hosts setup from the Networks tab
  • Auto-detects if hosts are already configured
  • Renders profile name, bio, and avatar as a styled page
  • CLI: pubky-node proxy-hosts <KEY>
🔑

Vanity Key Generator

Generate PKARR keypairs with a custom z-base-32 prefix or suffix. Uses all CPU cores for multi-threaded brute-force grinding.

  • Live input validation against z-base-32 alphabet
  • Progress bar with keys/s rate and time estimate
  • 3 chars = instant, 4 = seconds, 5 = minutes, 6+ = hours
  • CLI: pubky-node vanity <PREFIX> [--suffix]
•

Identity Watchlist

Monitor public keys and keep their records alive on the DHT by periodically re-publishing them. No secret key needed — this works with public keys only.

  • Add/remove keys from the Keys tab or CLI (pubky-node watchlist)
  • Keys are persisted to disk and survive restarts
  • Freshness check skips republish if record is still recent
  • Adjustable republish interval (default: 4 hours)
  • Great for keeping friend / service identities available
🖥

Local DNS Resolver

Resolve sovereign .pkarr domains on your local machine. Runs pkdns as a subprocess that intercepts DNS queries and resolves them via the DHT.

  • Disable with --no-dns if pkdns is not installed
  • Configurable ICANN fallback DNS server
  • Custom listen socket (default: 127.0.0.1:53)
  • Requires pkdns binary in PATH or configured via [dns] binary
🔒

Key Vault

Encrypted key storage protected by a password. Keys are encrypted using argon2id key derivation and ChaCha20-Poly1305 AEAD, stored in keyvault.enc.

  • Create, unlock, and lock your vault from the Keys tab
  • Store generated, vanity, imported, and homeserver keys
  • Export individual keys or full vault backup
  • Import keys from backup (duplicates are skipped)
  • Export to Pubky Ring app via pubkyring:// QR codes
Server

Homeserver

Built-in pubky-homeserver management with embedded PostgreSQL. No external database installation required — PostgreSQL is bundled and auto-managed.

  • Embedded PostgreSQL: auto-extracts on first run, port 5433
  • Auto-generates homeserver config with smart defaults
  • Start/stop server with real-time log streaming (SSE)
  • User management: list users, disable/enable, set storage quotas
  • Generate signup invite tokens with QR codes
  • Auto-publishes PKARR record when homeserver key is in vault
  • CLI: pubky-node homeserver status|start|stop|check|token|users|logs
🪪

Identity Manager

Sign up keys from your vault on the local homeserver using the EdDSA AuthToken protocol. Manage multiple identities from the Homeserver tab.

  • Sign up any vault key on your homeserver
  • Optional signup token for invite-only servers
  • Identity list persisted to disk
  • Sign in to refresh/verify existing sessions
☁️

Cloudflare Tunnels

Expose services to the public internet via cloudflared quick-tunnels — zero-config, no Cloudflare account needed.

  • Homeserver Tunnel — exposes the ICANN HTTP endpoint via trycloudflare.com
  • Relay Tunnel — exposes the Pkarr relay HTTP API
  • DNS Tunnel (DoH) — exposes pkdns as a public DNS-over-HTTPS endpoint at /dns-query
  • Auto-detects cloudflared binary (bundled sidecar → PATH)
  • Tunnel URLs auto-update homeserver config and PKARR record
  • Start/stop from Networks tab or CLI
💾

Backup & Recovery

Back up identity data from your homeserver. Syncs files locally with point-in-time snapshots for rollback protection.

  • Add identities to backup from the Recovery tab
  • Automatic periodic sync with manual force-sync option
  • Create snapshots for point-in-time recovery
  • Restore from any snapshot to revert compromised data
  • Export backups to portable archives
  • CLI: pubky-node backup status|sync-all|snapshot|snapshots
🚚

Homeserver Migration

Migrate your data to a different homeserver. Creates a pre-migration snapshot, uploads all data to the new server, and updates your PKARR record.

  • Preflight check validates target server connectivity and signup
  • Uses backup data or specific snapshot as migration source
  • Automatically updates PKARR record to point to new server
  • CLI: pubky-node migration preflight|execute|status
👤

Profile Management

Edit your public profile (name, bio, image) stored on your homeserver. Profiles are visible on pubky.app and indexed by Nexus.

  • Select an identity key to edit from the Profile tab
  • Check Nexus indexing status for discoverability
  • Verify PKARR record is correctly published
  • CLI: pubky-node profile get|set|nexus|verify
🌍

DNS Browser Setup

Point your browser at the local pkdns resolver to browse .pkarr and .key domains. pkdns forwards normal DNS queries to 8.8.8.8, so regular browsing is unaffected.

macOS (recommended — per-TLD, no system DNS change)

# Route only .pkarr and .key queries to local pkdns
sudo mkdir -p /etc/resolver
echo "nameserver 127.0.0.1" | sudo tee /etc/resolver/pkarr
echo "nameserver 127.0.0.1" | sudo tee /etc/resolver/key

Only sovereign TLDs are routed locally. All other DNS is untouched.

Linux (systemd-resolved)

# Create /etc/systemd/resolved.conf.d/pubky.conf
[Resolve]
DNS=127.0.0.1
Domains=~pkarr ~key

# Then restart:
sudo systemctl restart systemd-resolved

Windows

1. Settings → Network & Internet → Wi-Fi/Ethernet
2. Click your connection → DNS server assignment → Edit
3. Set Preferred DNS to: 127.0.0.1
4. Set Alternate DNS to: 8.8.8.8 (fallback)
5. Save

All Browsers: Disable Secure DNS

Browsers with "Secure DNS" (DoH) bypass your local resolver. Disable it:

  • Chrome/Edge: Settings → Privacy & Security → Use Secure DNS → Off
  • Firefox: Settings → Privacy → DNS over HTTPS → Off

Test It

# Verify pkdns is resolving:
dig @127.0.0.1 yg4gxe7z1r7mr6orids9fh95y7gxhdsxjqi6nngsxxtakqaxr5no.pkarr

CLI Quick Reference

## Running the Node
pubky-node                                    # Run with defaults
pubky-node run --no-dns                       # Skip DNS resolver
pubky-node run --verbose                      # Debug logging
pubky-node run --relay-port 8080              # Custom relay port
pubky-node run --dashboard-port 3000          # Custom dashboard port
pubky-node run --data-dir ./my-data           # Custom data directory

## DHT / Key Operations (no running node needed)
pubky-node resolve <KEY>                      # Resolve a key's DNS records
pubky-node resolve <KEY> --json               # Output as JSON
pubky-node publish --secret-key <HEX> \
  --record "HTTPS _pubky . target=host.com"   # Publish DNS records
pubky-node keygen                             # Generate new Ed25519 keypair
pubky-node keygen --json                      # Output keypair as JSON

## Vanity Keys
pubky-node vanity abc                         # Find key starting with "abc"
pubky-node vanity xyz --suffix                # Find key ending with "xyz"
pubky-node vanity pk -t 8                     # Use 8 threads

## HTTP Proxy
pubky-node proxy-hosts <KEY1> <KEY2>          # Add keys to /etc/hosts
pubky-node proxy-hosts --reset                # Remove proxy entries

## System / DNS
pubky-node status                             # Query running node status
pubky-node status --json                      # Raw JSON output
pubky-node dns-setup                          # Configure OS DNS resolver
pubky-node dns-setup --remove                 # Remove DNS configuration
pubky-node dns-setup --dry-run                # Preview changes

## Identity Watchlist (requires running node)
pubky-node watchlist list                     # Show all watched keys
pubky-node watchlist add <KEY>                # Add key to watchlist
pubky-node watchlist remove <KEY>             # Remove key from watchlist

## Homeserver (requires running node)
pubky-node homeserver status                  # State, PID, ports
pubky-node homeserver start                   # Start homeserver process
pubky-node homeserver stop                    # Stop homeserver process
pubky-node homeserver check                   # Prerequisites check
pubky-node homeserver token                   # Generate signup invite token
pubky-node homeserver users                   # List registered users
pubky-node homeserver publish-pkarr           # Publish PKARR record for homeserver key
pubky-node homeserver logs -n 100             # Tail homeserver logs

## Homeserver Tunnel (requires running node)
pubky-node tunnel status                      # Tunnel state + URL
pubky-node tunnel start                       # Start Cloudflare quick-tunnel
pubky-node tunnel stop                        # Stop quick-tunnel
pubky-node tunnel check                       # Check cloudflared binary

## Relay Tunnel (requires running node)
pubky-node relay-tunnel status                # Relay tunnel state + URL
pubky-node relay-tunnel start                 # Start relay tunnel
pubky-node relay-tunnel stop                  # Stop relay tunnel

## DNS Tunnel / DoH (requires running node)
pubky-node dns-tunnel status                  # DNS tunnel state + DoH URL
pubky-node dns-tunnel start                   # Start DNS-over-HTTPS tunnel
pubky-node dns-tunnel stop                    # Stop DNS tunnel

## Key Vault (requires running node)
pubky-node vault status                       # Vault lock state
pubky-node vault create --vault-password PWD  # Create new vault
pubky-node vault unlock --vault-password PWD  # Unlock vault
pubky-node vault lock                         # Lock vault
pubky-node vault keys                         # List all keys
pubky-node vault generate --label "My Key"    # Generate random keypair
pubky-node vault add --secret HEX             # Add existing secret
pubky-node vault export PUBKEY                # Export key secret
pubky-node vault export-all                   # Export all keys as JSON
pubky-node vault import keys.json             # Import keys from file
pubky-node vault rename PUBKEY "New Label"    # Rename key
pubky-node vault delete PUBKEY                # Delete key

## Identity (requires running node)
pubky-node identity list                      # List registered identities
pubky-node identity signup --pubkey KEY       # Sign up on homeserver
pubky-node identity signin --pubkey KEY       # Sign in (refresh session)

## Profile (requires running node)
pubky-node profile get PUBKEY                 # View profile
pubky-node profile set PUBKEY --name "Name"   # Update profile
pubky-node profile nexus PUBKEY               # Check Nexus indexing
pubky-node profile verify PUBKEY              # Verify PKARR record

## Backup & Recovery (requires running node)
pubky-node backup status                      # Backup status summary
pubky-node backup start --pubkey KEY          # Start backing up an identity
pubky-node backup stop --pubkey KEY           # Stop backing up an identity
pubky-node backup sync-all                    # Force sync all backups now
pubky-node backup snapshot                    # Create point-in-time snapshot
pubky-node backup snapshots                   # List available snapshots
pubky-node backup restore SNAPSHOT            # Restore from snapshot
pubky-node backup delete-snapshot SNAPSHOT    # Delete a snapshot

## Migration (requires running node)
pubky-node migration preflight \              # Preflight check
  --pubkey KEY --target URL
pubky-node migration execute \                # Execute migration
  --pubkey KEY --target URL
pubky-node migration status                   # Check migration progress

## Node Control (requires running node)
pubky-node node restart                       # Restart the node
pubky-node node shutdown                      # Graceful shutdown

## Auth: All commands support --password or PUBKY_NODE_PASSWORD env var
